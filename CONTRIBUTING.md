# Contributing

Open an issue before proposing a material change to the design-session allocation contract. Small corrections can go directly to a pull request.

A contribution must:

- preserve the conditional, visual-deliverable scope;
- keep non-visual Claude engineering outside the skill;
- retain design-native rendering and screenshot iteration;
- leave test inspection, engineering assurance and release proof to preparation or closure;
- cite a primary source for new factual model guidance;
- keep `SKILL.md` concise and move conditional detail into a one-level reference;
- pass `./scripts/validate.sh`.

Do not add generic prompt collections, model comparisons, design-system boilerplates, framework-specific implementation recipes or internal evaluation transcripts.
